# /project — 跨会话项目管理 Skill

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![Claude Code](https://img.shields.io/badge/Claude%20Code-Skill-orange)](https://claude.ai/code)

**做好分步骤计划，随时打开新会话，一条命令继续推进。不受时间和会话限制。**

---

## ⚡ 每次会话的标准流程

**每次会话就两个动作：开头 `/project`，收尾 `/project`。中间随意。**

```
会话开头                                会话收尾
────────                                ────────
   │                                      │
   ▼                                      ▼
/project                               /project
   │                                      │
   ▼                                      ▼
看进度 + 菜单                          选操作：
   │                               ┌─ 完成 N（步骤做完）
   ▼                               │
选「继续 N」                        └─ 保存 N（做了一部分，存断点）
   │
   ▼
工作（编码、讨论、决策……）
   │
   └────────── 过程中随时 /project 保存断点 ──────────┘
```

> **为什么必须 `/project`？** Claude Code 的 skill 必须显式输入 `/name` 才会加载指令。每次输入 `/project`，skill 指令被新鲜注入上下文——不会被长会话压缩影响。开头用它看进度，收尾用它保存，这是唯一可靠的方式。

---

## 暗号速查

每次 `/project` 后展示菜单，输入暗号操作：

| 暗号 | 作用 | 示例 |
|------|------|------|
| `继续 N` | 开始/继续第N步（有断点注入断点） | `继续 3` |
| `完成 N` | 第N步全部做完 → 标记done → 清checkpoint → 检查解锁 | `完成 3` |
| `保存 N` | 第N步做了一部分 → 审视对话 → 生成断点 → 确认 → 保存 | `保存 3` |
| `换项目` | 切换到其他项目（有进行中步骤会提醒） | `换项目` |
| `归档` | 当前项目移入 archive/（二次确认） | `归档` |
| `新建` | 创建新项目 | `新建` |

## 为什么需要这个？

Claude Code 没有跨会话记忆。每次新会话，Claude 从零开始——不知道你之前做了什么、哪些步骤已完成、卡在哪里、产出过什么结论。

大部分需要多步骤推进的事情，都不是一次会话能做完的。今天做一部分，明天接着来。如果每次都要重新描述背景、回忆进度、翻聊天记录找上次的产出——这件事本身就消耗了你最宝贵的注意力。

`/project` 解决的就是这个：**做好分步骤计划，随时打开新会话，一条命令继续推进。** 不是记笔记，不是 Todo List——是给 AI 装了一个外置的项目记忆。

## 快速开始

### 安装

```bash
mkdir -p ~/.claude/skills/project
cp SKILL.md ~/.claude/skills/project/
```

或使用初始化脚本：

```bash
bash scripts/init.sh
```


## 能力与边界

| ✅ 能做到 | ❌ 做不到 |
|-----------|----------|
| `/project` 唤起 → 读 YAML → 恢复项目上下文 + 展示菜单 | 在你说 `/project` 之前自动感知"该续接了" |
| `保存 N` → 审视会话对话 → 提炼断点草稿 → 你确认后保存 | 记住你没写进 YAML 或产出文件的讨论细节 |
| `继续 N` → 自动注入上次断点为工作上下文 | 自动判断产出质量是否合格 |
| `完成 N` → 检测产出文件是否存在 → 标记完成 | 跨项目自动感知关联 |
| DAG 依赖检测 + 步骤解锁 | 在你忘记 `/project` 时强制触发 |

**核心承诺：每次 `/project`，就能在中断后的新会话里恢复项目全貌——做到哪了、断了什么、还剩什么、该做什么。**

## 核心卖点

### 1. DAG 编排 + 步骤内断点

```
第1步 现状分析 ────────────────────────┐
第2步 组件抽离 ──→ 第5步 状态迁移 ─────┤
第3步 用户模块 ───────────────────────┤  第3步内部：
第4步 报表模块 ───────────────────────┘    会话A: ✅ 列表页 ⬜ 详情页/权限
                                           ↓      保存 3 → 断点写入
                                   第6步 回归测试  会话B: 继续 3 → 断点注入
                                                      ✅ 列表页+详情页 ⬜ 权限
                                                      保存 3 → 断点更新
                                                      会话C: 继续 3 → ...
                                                      完成 3 → checkpoint 清除
```

### 2. 暗号操作，不靠自然语言

自然语言在长会话中不可靠——指令可能被压缩丢失。`/project` 每次新鲜加载，暗号短、无歧义、不会被漏掉。

| 暗号 | 含义 |
|------|------|
| `继续 N` | 开始工作 |
| `完成 N` | 步骤做完 |
| `保存 N` | 做了一部分，存断点 |

### 3. 零子命令，一条指令

不区分 `/project start`、`/project save`、`/project done`。就 `/project`，看完菜单选暗号。

### 4. 多项目隔离

```
.claude/projects/
  admin-refactor.yaml
  blog-series.yaml
  archive/
    old-feature.yaml
```

`/project` 只有一个活跃项目时直接加载，多个时列出让你选。`换项目` 时如果有进行中步骤会提醒。

### 5. 上下文分级注入

| 级别 | 加载内容 | 触发条件 |
|------|---------|---------|
| L1 | `context` 字段（200 字项目背景） | 每次 `/project` |
| L2 | done 步骤摘要 + in_progress 步骤断点 | 每次 `/project` |
| L3 | 产出文件全文 | "看看 XX 产出"时 |

### 6. 零依赖

一个 `SKILL.md` + `.claude/projects/*.yaml`。不需要 MCP Server、API key、外部服务。

## 常见问题

### 为什么每次会话开头和收尾都要 `/project`？

不是我们想这样——是 Claude Code 的机制决定的。Skill 指令只会因为 `/name` 注入上下文。开头 `/project` 看进度，收尾 `/project` 保存——两次 `/project` 都是新鲜加载指令，不会被长会话压缩影响。

### `/project` 和 CLAUDE.md 的 auto-memory 有什么区别？

CLAUDE.md 存知识（"用了什么库、决定了什么架构"）。`/project` 存进程（"做到第几步了、断了哪些、还剩哪些"）。两者互补。

### checkpoint 和 summary 有什么区别？

| | checkpoint | summary |
|---|---|---|
| **何时存在** | 步骤 in_progress 期间 | 步骤 done 之后 |
| **目的** | 续接——"从哪继续" | 回顾——"做了什么" |
| **生成** | `保存 N` 时审视对话 + 你确认 | `完成 N` 时你一句话总结 |
| **生命周期** | `完成 N` 时自动清除 | 永久保留 |

### 怎么归档项目？

在 `/project` 菜单下输入 `归档`，二次确认后移入 `archive/`。可随时手动移回。

### 上下文会爆吗？

不会。L1（200字 context）+ L2（每步一行摘要 + checkpoint 摘要）。产出全文只在你说"看看"时才加载。
